### Task 1: Campaign Dashboard sub-command (GADS-01)

**File**: `claude-ops/skills/ops-marketing/SKILL.md` → `## google-ads` section

Add sub-command routing table immediately after the credential/token guard:

| Input | Action |
|---|---|
| (empty), dashboard, overview | Campaign dashboard — last 7 days |
| search-terms, terms | Search Terms Report — last 30 days |
| budget-recs, recommendations, recs | Budget recommendations |
| campaigns, manage | Plan 03 |
| keywords, kw | Plan 03 |

Implement `### dashboard` sub-command:

**GAQL query** (POST to `https://googleads.googleapis.com/v23/customers/${GADS_CUSTOMER_ID}/googleAds:searchStream`):
```
SELECT campaign.id, campaign.name, campaign.status, campaign_budget.amount_micros,
  metrics.cost_micros, metrics.impressions, metrics.clicks, metrics.conversions,
  metrics.conversions_value
FROM campaign
WHERE segments.date DURING LAST_7_DAYS AND campaign.status != REMOVED
ORDER BY metrics.cost_micros DESC LIMIT 20
```

**Output format** (banner + summary header + per-campaign table):
- Banner: `══ GOOGLE ADS — Last 7 Days ══`
- Summary: Total Spend (`SUM(costMicros)/1e6` via awk), Total Conversions, Overall ROAS (`conversionsValue/spend` with zero-spend guard showing "—")
- Per-campaign table: `Campaign | Status | Budget/day | Spend | Impr | Clicks | CTR | Conv | ROAS`
- All monetary values: `costMicros / 1000000` via `awk printf "%.2f"`
- CTR: `clicks / impressions * 100` with zero-impressions guard
- Empty state: `No active campaigns found in the last 7 days.`
- Error: parse `.[0].error.message`, print with credential hint

**Implementation notes:**
- Use `jq @tsv + while IFS=$'\t' read` pipeline to avoid subshell output loss
- Render table using `printf "%-40s %-10s ..."` column format

### Task 2: Search Terms Report and Budget Recommendations (GADS-04, GADS-06)

**File**: `claude-ops/skills/ops-marketing/SKILL.md` → `## google-ads` section

**Search Terms sub-command** (GADS-04):

GAQL query on `search_term_view` resource:
```
SELECT search_term_view.search_term, search_term_view.status,
  campaign.name, ad_group.name,
  metrics.impressions, metrics.clicks, metrics.cost_micros, metrics.conversions
FROM search_term_view
WHERE segments.date DURING LAST_30_DAYS AND metrics.impressions > 0
ORDER BY metrics.impressions DESC LIMIT 100
```

Status column mapping: `ADDED` → `✓ Added`, `EXCLUDED` → `✗ Excluded`, `NONE` → `○ New`

Per-term table: `Search Term | Status | Campaign | Ad Group | Impr | Clicks | Cost | Conv`

Negative keyword candidates section using inline `jq select()`:
- Filter: `conversions == 0 AND costMicros > 1000000` (>$1 spend, no conversions)
- Print as bullet list under `Negative keyword candidates:`

Empty state: `No search term data found for the last 30 days.`

**Budget Recommendations sub-command** (GADS-06):

GAQL query on `recommendation` resource:
```
SELECT recommendation.type, recommendation.campaign, recommendation.text,
  recommendation.campaign_budget_recommendation.current_budget_amount_micros,
  recommendation.campaign_budget_recommendation.recommended_budget_amount_micros,
  recommendation.campaign_budget_recommendation.base_metrics.impressions,
  recommendation.campaign_budget_recommendation.potential_metrics.impressions
FROM recommendation
WHERE recommendation.type IN (
  CAMPAIGN_BUDGET, MOVE_UNUSED_BUDGET,
  MARGINAL_ROI_CAMPAIGN_BUDGET, FORECASTING_CAMPAIGN_BUDGET
)
```

Type column human-readable labels:
- `CAMPAIGN_BUDGET` → `Budget Increase`
- `MOVE_UNUSED_BUDGET` → `Reallocate Budget`
- `MARGINAL_ROI_CAMPAIGN_BUDGET` → `Marginal ROI`
- `FORECASTING_CAMPAIGN_BUDGET` → `Forecast Boost`

Per-recommendation table: `Type | Campaign | Current Budget | Recommended | Impact`

Impact: percentage impressions increase `(potentialMetrics.impressions - baseMetrics.impressions) / baseMetrics.impressions * 100`; show `—` if base is 0.

Empty state: `No budget recommendations available. Google needs campaign data to generate recommendations.`
