## Task Breakdown

### Task 1: Campaign Management sub-commands (GADS-02)

**File**: `claude-ops/skills/ops-marketing/SKILL.md` → `## google-ads` section

Replace placeholder routing rows with:

| Input | Action |
|---|---|
| campaigns, manage | Campaign management — list, create, pause, enable, adjust budget |
| keywords, kw, keyword-planner | Keyword Planner — discover keywords with volume and bid data |
| ad-groups, ag | Ad group management — list, create, add/remove keywords, adjust bids |

Implement `### campaigns` sub-section:

**List campaigns** (`campaigns` / `campaigns list`):
- GAQL `searchStream` selecting `campaign.id, campaign.name, campaign.status, campaign.advertising_channel_type, campaign_budget.amount_micros`
- Output as banner + table: `ID | Name | Status | Type | Budget/day`

**Create campaign** (`campaigns create`):
- AskUserQuestion: campaign name, daily budget (USD), channel type (SEARCH/DISPLAY/SHOPPING/VIDEO)
- Two-step mutate:
  1. POST to `campaignBudgets:mutate` — create budget resource, capture `results[0].resourceName`
  2. POST to `campaigns:mutate` — create campaign with `status: PAUSED` (always starts paused for safety), linking the budget resource name
- Print created campaign ID; note that user must manually enable after review

**Pause campaign** (`campaigns pause <ID>`):
- AskUserQuestion confirmation (Rule 5) showing campaign name + ID
- POST to `campaigns:mutate` with `updateMask: "status"`, `status: PAUSED`

**Enable campaign** (`campaigns enable <ID>`):
- No confirmation needed (non-destructive)
- POST to `campaigns:mutate` with `updateMask: "status"`, `status: ENABLED`

**Adjust budget** (`campaigns budget <ID> <AMOUNT_USD>`):
- Fetch current budget resource name via `searchStream` on `campaign_budget` filtered by campaign ID
- AskUserQuestion confirmation (Rule 5) showing old amount → new amount
- POST to `campaignBudgets:mutate` with `updateMask: "amountMicros"`, converting USD to micros (`amount * 1000000`)

### Task 2: Keyword Planner and Ad Group Management (GADS-03, GADS-05)

**File**: `claude-ops/skills/ops-marketing/SKILL.md` → `## google-ads` section

**Keyword Planner sub-section** (GADS-03):

- Collect seed keywords via AskUserQuestion (comma-separated, split to JSON array)
- POST to `:generateKeywordIdeas` on customer resource
- Defaults: `languageConstants/1000` (English), `geoTargetConstants/2840` (United States); document alternatives for UK/Canada/Australia
- Output table: `Keyword | Avg Monthly Searches | Competition | Low Bid | High Bid`
  - `lowTopOfPageBidMicros` and `highTopOfPageBidMicros` ÷ 1,000,000 → `$X.XX`
  - Sorted by `avgMonthlySearches` descending
- Empty state: `No keyword ideas found for these seeds. Try different or broader keywords.`

**Ad Group Management sub-section** (GADS-05):

- **List ad groups** (`ad-groups list <CAMPAIGN_ID>`): GAQL query filtered by `campaign.id`, status != REMOVED
- **Create ad group** (`ad-groups create <CAMPAIGN_ID>`):
  - AskUserQuestion: name, CPC bid (USD)
  - POST to `adGroups:mutate` with `type: SEARCH_STANDARD`, bid converted to micros
- **List keywords** (`ad-groups keywords <AD_GROUP_ID>`): GAQL on `ad_group_criterion` where type = KEYWORD
- **Add keyword** (`ad-groups add-keyword <AD_GROUP_ID>`):
  - AskUserQuestion: keyword text, match type (Broad/Phrase/Exact → BROAD/PHRASE/EXACT), optional CPC bid
  - POST to `adGroupCriteria:mutate` with create operation
  - Use bash conditional expansion for optional bid field
- **Remove keyword** (`ad-groups remove-keyword <AD_GROUP_ID> <CRITERION_ID>`):
  - AskUserQuestion confirmation (Rule 5)
  - POST to `adGroupCriteria:mutate` with remove operation, resource name `adGroupCriteria/{AD_GROUP_ID}~{CRITERION_ID}`
- **Update bid** (`ad-groups update-bid <AD_GROUP_ID> <CRITERION_ID> <BID_USD>`):
  - POST to `adGroupCriteria:mutate` with `updateMask: "cpcBidMicros"` only
  - Document keyword immutability: text and match type cannot be changed; require remove+recreate
