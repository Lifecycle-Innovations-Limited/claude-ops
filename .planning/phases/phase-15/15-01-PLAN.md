# Phase 15 Plan 01: Cross-Platform Marketing Intelligence

## Goal

Stitch together the per-platform integrations built in Phases 10–14 into a unified intelligence layer: a new `marketing-optimizer` sub-agent (claude-sonnet model) that computes blended ROAS and recommends budget shifts, a cross-platform `attribution` sub-command, a unified `campaigns` view across Google Ads + Meta + Klaviyo, a blended ROAS + health score in the dashboard script, and a MARKETING section in the `/ops:go` morning briefing.

**Prerequisite**: Phases 10–14 must be complete (credential blocks and gather functions in place).

## Requirements Addressed

- Cross-platform marketing intelligence synthesis
- Blended ROAS computation across Google Ads + Meta Ads
- Budget shift recommendations based on platform performance differential
- Morning briefing integration via `/ops:go`

## Runtime Prerequisites

No new credentials required. This phase assembles data from channels already configured in Phases 10–14. All channel data is optional — missing channels show as `null` in output, and the optimizer gracefully handles partial configurations.

## Task Breakdown

### Task 1: Create marketing-optimizer agent

**File**: `claude-ops/agents/marketing-optimizer.md` (new)

YAML frontmatter:
```yaml
---
name: marketing-optimizer
description: Cross-platform ad budget optimization — reads Meta + Google Ads data, computes blended ROAS, and recommends specific budget shifts.
model: sonnet
---
```

**Agent instructions:**

Purpose: analyze ad performance across Meta Ads and Google Ads, compute blended ROAS, produce actionable budget recommendations.

**Input**: read pre-gathered data from ops-marketing-dash:
```bash
"${CLAUDE_PLUGIN_ROOT:-$HOME/.claude/plugins/ops-ops-marketplace}/bin/ops-marketing-dash" 2>/dev/null
```

**JSON schema** that ops-marketing-dash v1.5+ emits (document fully):
- `meta` — account totals (last 7 days): `spend`, `impressions`, `clicks`, `ctr`, `purchases`, `purchase_value`, `roas` (all strings)
- `google_ads` — raw searchStream response (array of pages) or `null` if unconfigured
- `klaviyo` — `subscribers`, `last_campaign`, `last_campaign_status`, `open_rate`
- `ga4` — `sessions`, `users`, `conversions`, `revenue`, `cvr`
- `gsc` — `clicks`, `impressions`, `ctr`, `avg_position`
- `instagram` — `followers`, `media_count`, `reach_7d`
- Top-level: `blended_roas`, `health_score`, `health_status`, `date`

**Data extraction helpers** using null-safe jq:
- Google Ads spend: `(if type=="array" then [.[].results[]?.metrics.costMicros // "0" | tonumber] | add // 0 else 0 end) / 1000000`
- Google Ads conversions_value: same pattern with `conversionsValue`

**Analysis logic** (agent applies this reasoning):
1. Compute platform ROAS: `meta.roas` (already computed), Google ROAS from raw data
2. Compute blended ROAS: `(meta_spend * meta_roas + google_spend * google_roas) / (meta_spend + google_spend)`
3. Identify imbalance: if one platform ROAS > 20% above blended, recommend shifting budget toward it
4. Threshold: only recommend shifts > $50/day; include specific dollar amounts
5. Consider channel saturation: large share of budget on one platform may hit diminishing returns

**Output format**: structured recommendation with:
- Blended ROAS summary
- Per-platform performance table
- Specific budget shift recommendations (if any): `Move $X/day from Meta → Google Ads (Meta ROAS: 1.8x, Google ROAS: 3.2x)`
- Warning if total paid spend < $100/day (insufficient data for recommendations)

### Task 3: Extend ops-marketing-dash with Instagram gather, blended ROAS, health score

**File**: `claude-ops/bin/ops-marketing-dash`

**Instagram gather function** (`gather_instagram`):
- Credential guard: if `META_ACCESS_TOKEN` or `INSTAGRAM_ACCOUNT_ID` empty, return `null`
- GET `/{INSTAGRAM_ACCOUNT_ID}?fields=followers_count,media_count&access_token=<TOKEN>`
- GET `/{INSTAGRAM_ACCOUNT_ID}/insights?metric=reach&period=day&since=7daysAgo&until=today`
- Aggregate `reach` sum over 7 days
- Return: `{"followers": N, "media_count": N, "reach_7d": N}`

Add `INSTAGRAM_DATA=$(gather_instagram) &` to parallel execution block.

**Blended ROAS computation** (after all gathers complete):
```bash
META_SPEND=$(echo "$META_DATA" | jq -r '.spend // "0"')
META_CONV_VALUE=$(echo "$META_DATA" | jq -r '.purchase_value // "0"')
GOOGLE_SPEND=$(echo "$GADS_DATA" | jq '[.[].results[]?.metrics.costMicros // "0" | tonumber] | add // 0' 2>/dev/null | awk '{printf "%.2f", $1/1000000}')
GOOGLE_CONV_VALUE=$(echo "$GADS_DATA" | jq '[.[].results[]?.metrics.conversionsValue // "0" | tonumber] | add // 0' 2>/dev/null)
TOTAL_SPEND=$(awk "BEGIN{printf \"%.2f\", $META_SPEND + $GOOGLE_SPEND}")
TOTAL_VALUE=$(awk "BEGIN{printf \"%.2f\", $META_CONV_VALUE + $GOOGLE_CONV_VALUE}")
BLENDED_ROAS=$(awk "BEGIN{if ($TOTAL_SPEND > 0) printf \"%.2f\", $TOTAL_VALUE / $TOTAL_SPEND; else print \"0\"}")
```

**Health score computation** (0–100 composite):
- ROAS trend (0–40 pts): blended ROAS ≥ 3.0 → 40, ≥ 2.0 → 30, ≥ 1.5 → 20, ≥ 1.0 → 10, < 1.0 → 0
- Spend efficiency (0–30 pts): CTR ≥ 2% → 30, ≥ 1% → 20, ≥ 0.5% → 10
- Channel diversification (0–30 pts): 3+ channels with spend → 30, 2 channels → 20, 1 channel → 10
- `health_status`: score ≥ 80 → `"EXCELLENT"`, ≥ 60 → `"GOOD"`, ≥ 40 → `"FAIR"`, < 40 → `"POOR"`

Add to final JSON output:
```json
{
  "instagram": ...,
  "blended_roas": "2.84",
  "health_score": 73,
  "health_status": "GOOD",
  "date": "..."
}
```
