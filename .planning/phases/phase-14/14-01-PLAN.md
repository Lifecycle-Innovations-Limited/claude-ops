# Phase 14 Plan 01: WhatsApp Business API

## Goal

Create a new `skills/ops-whatsapp-biz/SKILL.md` skill for the WhatsApp Business Cloud API (Meta Graph API), distinct from `wacli` personal WhatsApp. Implement approved template sends (single + bulk), template management (list/create/check status), product catalog viewer, and a guided setup wizard sub-flow integrated into `/ops:setup`. This is business-to-customer messaging at scale — not personal chat.

## Requirements Addressed

- WhatsApp Business Cloud API integration for outbound marketing and transactional messages
- Template management lifecycle (create → approval → send)
- Product catalog integration

## Runtime Prerequisites

**STOP — the following credentials must be configured before this plan can execute:**

| Credential | Where to get it |
|---|---|
| `WABA_TOKEN` | Meta Business Manager → WhatsApp → API Setup → Access Token (Permanent System User token recommended; temp tokens expire in 24h) |
| `WABA_PHONE_ID` | Meta Business Manager → WhatsApp → API Setup → Phone Number ID (numeric) |
| `WABA_ACCOUNT_ID` | Meta Business Manager → WhatsApp → Overview → WhatsApp Business Account ID |

**Account requirements:**
- WhatsApp Business Account (WABA) verified with Meta
- Phone number verified and registered in the WABA
- For production sends: business verification completed with Meta
- Templates must be pre-approved by Meta (2-24h review time) before sending

**Pricing note:** WhatsApp Business API charges per conversation, not per message. Marketing conversations: ~$0.0125/conversation (US). Display pricing note on send operations.

**Separate from wacli:** This skill uses the WhatsApp Business Cloud API (`graph.facebook.com/v19.0/{phone-number-id}/messages`). The personal `wacli` uses whatsmeow bridge. Different credentials, different phone numbers, different use cases. Never conflate the two.

```bash
PREFS_PATH="${CLAUDE_PLUGIN_DATA_DIR:-$HOME/.claude/plugins/data/ops-ops-marketplace}/preferences.json"

# Doppler fallbacks
[ -z "$WABA_TOKEN" ] && WABA_TOKEN="$(doppler secrets get WHATSAPP_BUSINESS_TOKEN --plain 2>/dev/null)"
[ -z "$WABA_PHONE_ID" ] && WABA_PHONE_ID="$(doppler secrets get WHATSAPP_PHONE_NUMBER_ID --plain 2>/dev/null)"
[ -z "$WABA_ACCOUNT_ID" ] && WABA_ACCOUNT_ID="$(doppler secrets get WHATSAPP_BUSINESS_ACCOUNT_ID --plain 2>/dev/null)"
# preferences.json fallbacks (written by setup wizard Task 5)
if [ -f "$PREFS_PATH" ]; then
  [ -z "$WABA_TOKEN" ] && WABA_TOKEN="$(jq -r '.whatsapp_business.token // empty' "$PREFS_PATH" 2>/dev/null)"
  [ -z "$WABA_PHONE_ID" ] && WABA_PHONE_ID="$(jq -r '.whatsapp_business.phone_number_id // empty' "$PREFS_PATH" 2>/dev/null)"
  [ -z "$WABA_ACCOUNT_ID" ] && WABA_ACCOUNT_ID="$(jq -r '.whatsapp_business.business_account_id // empty' "$PREFS_PATH" 2>/dev/null)"
fi
```

### Task 2: Template Listing and Sending

**File**: `claude-ops/skills/ops-whatsapp-biz/SKILL.md`

**List Templates** (`list-templates` / empty):

GET `https://graph.facebook.com/v19.0/{WABA_ACCOUNT_ID}/message_templates?fields=name,status,category,language,components&access_token={WABA_TOKEN}`

Output table:
```
══ WHATSAPP TEMPLATES ══
Name                  Category    Language  Status
order_confirmation    UTILITY     en        ✓ APPROVED
promo_offer           MARKETING   en        ✗ REJECTED
shipping_update       UTILITY     nl        … PENDING
```

Status mapping: `APPROVED` → `✓ APPROVED`, `REJECTED` → `✗ REJECTED`, `PENDING` → `… PENDING`, `DELETED` → `— DELETED`

Pricing note printed after table: `Note: WA Business API charges per conversation (~$0.0125 marketing, ~$0.005 utility, US).`

**Send Template** (`send-template`):

AskUserQuestion (multi-step):
- Step 1: template name (must be in APPROVED state)
- Step 2: recipient phone number(s) — single number or comma-separated list for bulk
- Step 3: template variable values (if template has `{{1}}`, `{{2}}` parameters — parse components from list-templates first)

For single send: POST to `/{WABA_PHONE_ID}/messages`:
```json
{
  "messaging_product": "whatsapp",
  "to": "<phone_e164>",
  "type": "template",
  "template": {
    "name": "<template_name>",
    "language": {"code": "en"},
    "components": [{"type": "body", "parameters": [{"type": "text", "text": "<value>"}]}]
  }
}
```

For bulk (comma-separated phones):
- AskUserQuestion confirmation (Rule 5): show recipient count + cost estimate (`count × $0.0125`)
- Loop sends with 100ms delay between each to avoid rate limits

Print message ID for each successful send.

### Task 3: Template Creation and Status Checking

**File**: `claude-ops/skills/ops-whatsapp-biz/SKILL.md`

**Create Template** (`create-template`) — guided wizard:

AskUserQuestion sequence:
- Template name (lowercase, underscores, max 512 chars)
- Category: UTILITY / MARKETING / AUTHENTICATION
- Language code (en / nl / es / de / fr — show common list)
- Header (optional): TEXT / IMAGE / VIDEO / DOCUMENT
- Body text (required): enter template text with `{{N}}` placeholders for variables
- Footer (optional): short text
- Buttons (optional): QUICK_REPLY / URL / PHONE_NUMBER

POST to `/{WABA_ACCOUNT_ID}/message_templates`:
```json
{
  "name": "<name>",
  "category": "<CATEGORY>",
  "language": "<code>",
  "components": [...]
}
```

Print template ID; note 2-24h review time. Run `check-template <name>` to monitor.

**Check Template Status** (`check-template <NAME>`):

GET `/{WABA_ACCOUNT_ID}/message_templates?name=<NAME>&fields=name,status,rejected_reason`

Print current status + reason if rejected:
```
Template: order_confirmation
Status:   ✓ APPROVED
Language: en
```
If REJECTED: print `Rejected reason: <reason>` and suggest edits.

## Threat Model
